# Topics Research

B2B 制造商博客话题研究工具，使用 Query Fan Out 策略生成行业通用话题。

## 核心功能

- **Query Fan Out**：从公司核心业务向 9 大维度扩展搜索
- **增量扩展**：已完成研究可随时增加更多话题
- **公司匹配前置**：先列公司列表，再生成高度匹配的话题
- **去重机制**：基于站点地图避免话题重复

## 快速开始

### 1. 为现有公司研究话题
```
请为 [公司名] 进行话题研究
```

### 2. 扩展已有话题
```
请为 [公司名] 增加更多 [方向] 话题
```

### 3. 新话题方向（公司列表前置）
```
我想研究 [话题方向] 相关话题
```
→ Claude 列出公司列表 → 你选择 → 生成匹配话题

### 4. 同步站点地图
```
请为 [公司名] 更新 posts-sitemap，格式为 xxx.com/post-sitemap.xml
```

## 9 大研究维度

| 维度 | 描述 |
|------|------|
| 材料科学 | 合金、规格、标准 |
| 质量控制 | 缺陷、检验、标准 |
| 应用场景 | 行业特定需求 |
| 材料对比 | 不同材料比较 |
| 成本分析 | 生产经济学 |
| 法规认证 | 合规要求 |
| 市场趋势 | 行业展望 |
| 可持续发展 | 环保、回收 |
| 生产效率 | 自动化、生产力 |

## 项目结构

```

├── companies/
│   ├── _template.md                  # 公司档案模板
│   ├── _posts-sitemap-template.md    # 站点地图模板
│   └── [公司名]/
│       ├── profile.md                # 公司档案
│       └── posts-sitemap.md          # 已发布文章站点地图
├── output/
│   └── [公司名]-topics-research.md   # 研究报告
├── WORKFLOW.md                       # 详细流程文档
└── README.md
```

## 去重机制

每个公司的 `posts-sitemap.md` 记录已发布文章，生成新话题时自动检查：

- 关键词不重复
- 搜索意图不重复
- 话题集群平衡覆盖

## 输出格式

- 话题分类表（按维度）
- 话题列表（含搜索意图、关键词）
- 优先级矩阵（Tier 1/2/3）
- SEO 关键词集群
- 话题集群覆盖分析

## 站点地图内容

| 字段 | 用途 |
|------|------|
| 文章索引 | 标题、搜索意图、关键词、URL |
| 意图统计 | Informational/Commercial/Transactional 数量 |
| 集群覆盖 | 各话题集群的已写/待写数量 |
| 已用关键词 | 生成新话题时避免重复 |
| 内容缺口 | 建议新增的话题方向 |
